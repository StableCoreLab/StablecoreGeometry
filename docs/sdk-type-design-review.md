# SDK 类型设计评审

## 1. 范围

本评审只回顾当前算法库公开类型设计、派生体系、类 / 结构体命名、方法位置与模块边界，不涉及实现正确性和性能细节。

## 2. 当前结论摘要

- 公开 API 命名继续使用短名风格
- 旧 `GetXxx` 风格不再扩散
- 成员方法与自由函数边界要继续明确
- 类型层不应承担复杂算法职责
- 2D 的收敛规则将继续约束未来 3D 设计

## 3. 评审重点

后续新增类型或调整类型时，应重点检查：

1. 类型语义是否清楚
2. 命名是否与现有 SDK 一致
3. 成员方法是否越界承担算法职责
4. 是否需要同步文档与测试
