# Geometry SDK 与二进制发布设计

## 1. 目的

本文定义几何库未来以 `DLL + import lib + headers` 形式发布时的 SDK 边界。

## 2. 设计目标

- 给调用方提供稳定、清晰的公开 API
- 控制 ABI 暴露范围
- 避免模板、内部 helper 与复杂对象状态直接泄漏给外部

## 3. 公开层建议

对外建议分成：

- 基础值类型头文件
- SDK 服务头文件
- 聚合入口头文件

## 4. 当前固定结论

- 公开 API 优先暴露稳定值类型与自由函数服务
- 复杂对象的 ABI 暴露要谨慎
- 内部实现细节不应直接进入 SDK 头文件
