# Point2 与 Vector2 设计

## 1. 目的

本文定义当前仓库中二维点与二维向量的类型语义、边界和实现约束。

## 2. 核心语义

- `Point2<T>` 表示位置
- `Vector2<T>` 表示位移 / 方向量
- 两者不能因为底层字段相同就混用语义

## 3. 设计原则

### 3.1 使用模板统一类型体系

建议以：

- `Point2<T>`
- `Vector2<T>`

作为基础模板，并通过别名形成：

- `Point2d`
- `Point2i`
- `Vector2d`
- `Vector2i`

### 3.2 类型层保持轻量

类型层适合提供：

- 构造
- 坐标访问
- 少量天然成立的运算

不适合直接承担：

- 复杂投影
- 多对象关系判定
- 大型算法流程

### 3.3 算法层默认以 `double` 为主

模板化类型系统是合理的，但复杂几何算法仍应以 `double` 计算为主，整数类型更多用于输入表达。

## 4. 当前固定结论

- 点与向量语义分离
- 保留模板化方向
- 复杂算法不塞回成员函数
- 命名继续保持短名风格
