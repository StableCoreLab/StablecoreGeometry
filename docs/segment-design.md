# 线段体系开发设计文档

## 1. 文档目的

本文档用于定义几何库中线段体系的开发方案。后续开发人员必须按本文档实现，不再按临时理解扩展。

本文档覆盖以下内容：

- 统一线段基类
- 直线段与弧线段的类型边界
- 公共接口
- 几何语义
- 算法层依赖方式
- 第一版实现范围

## 2. 设计目标

线段体系必须满足以下目标：

- 对上层算法提供统一的段类抽象
- 同时支持直线段和弧线段
- 保持公共接口一致
- 允许算法先做类型分发，再进入具体实现
- 为未来折线、轮廓、区域边界表达打基础

这里的“线段”不是只指直线段，而是指二维轮廓中的“边段单元”。

因此第一版要支持两类边段：

- 直线段
- 圆弧段

## 3. 总体结构

必须采用统一基类 + 派生类的结构。

第一版按以下命名实现：

```cpp
template <typename T>
class Segment;

template <typename T>
class LineSegment;

template <typename T>
class ArcSegment;
```

并提供常用别名：

```cpp
using Segmentd = Segment<double>;
using LineSegmentd = LineSegment<double>;
using ArcSegmentd = ArcSegment<double>;
```

说明：

- 基类负责统一语义与公共接口
- 直线段和弧线段负责各自数据结构和具体行为
- 后续算法优先面向基类接口编程

## 4. 为什么必须有统一基类

统一基类不是为了面向对象形式感，而是为了后续算法和数据结构稳定。

后续这些对象都会依赖“边段单元”抽象：

- 轮廓
- 路径
- 构件边界
- 算量边界线

如果没有统一基类，后续调用方会很快出现这些问题：

- 轮廓容器无法统一承载直线段和弧线段
- 投影、包围盒、端点提取等逻辑要重复写
- 算法层接口会被迫拆成多套

因此第一版必须先统一“边段抽象”。

## 5. 类型体系

### 5.1 `Segment<T>`

`Segment<T>` 是统一边段抽象基类。

它的职责是：

- 对外表达“这是一个二维边段”
- 提供所有边段都必须具备的公共接口
- 作为算法层统一入口

它不负责：

- 保存具体的直线段数据细节
- 保存具体的弧线段数据细节
- 实现派生类专属算法

### 5.2 `LineSegment<T>`

`LineSegment<T>` 表示有限直线段。

它负责：

- 保存起点和终点
- 实现直线段专属的长度、方向、包围盒等行为

### 5.3 `ArcSegment<T>`

`ArcSegment<T>` 表示圆弧段。

它负责：

- 保存圆弧参数
- 实现弧长、起终点、包围盒等行为

第一版中的弧线仅定义为圆弧，不引入椭圆弧或样条曲线。

## 6. 段类型枚举

统一基类必须暴露段类型。

必须定义：

```cpp
enum class SegmentKind
{
    Line,
    Arc
};
```

`Segment<T>` 必须提供：

- `GetKind()`

执行要求：

- 后续算法允许先看 `GetKind()` 再分派
- 不允许调用方依赖隐式假设去猜对象真实类型

## 7. 基类必须提供的公共接口

`Segment<T>` 必须至少提供以下公共接口。

### 7.1 类型接口

- `GetKind()`

### 7.2 端点接口

- `GetStartPoint()`
- `GetEndPoint()`

所有边段都必须有明确起点和终点。

### 7.3 长度接口

- `Length()`
- `LengthSquared()` 可选

说明：

- `Length()` 必须提供
- `LengthSquared()` 对直线段自然，但对弧段意义不大，因此第一版可不做统一强制

### 7.4 包围盒接口

- `GetBoundingBox()`

这是后续粗判、快速过滤和空间索引的基础接口，必须统一。

### 7.5 参数接口

- `GetPointAt(parameter)`

参数区间统一定义为：

- `parameter` 取值范围 `[0, 1]`

统一语义：

- `0` 对应起点
- `1` 对应终点

该接口对算法层非常关键，因为后续投影、最近点、交点位置都会落到参数表达上。

### 7.6 有效性接口

- `IsValid()`

执行要求：

- 无效几何对象必须能被显式识别
- 不能把退化输入默默当有效对象处理

## 8. 基类禁止承载的内容

`Segment<T>` 禁止承载以下内容：

- 直线段专属方向向量缓存
- 弧线段专属圆心、半径、角度字段
- 投影算法细节
- 求交算法细节
- 与业务对象耦合的字段

原因：

- 基类只负责统一抽象，不负责具体实现细节

## 9. `LineSegment<T>` 数据要求

`LineSegment<T>` 必须至少保存：

- 起点
- 终点

第一版不要求额外缓存方向、长度等派生数据。

### 9.1 必须满足的规则

- 起点和终点必须可访问
- 零长度直线段必须能识别为退化对象

### 9.2 可提供的专属能力

`LineSegment<T>` 可以额外提供：

- `Direction()`
- `DirectionVector()`

但这些不进入统一基类接口。

## 10. `ArcSegment<T>` 数据要求

第一版 `ArcSegment<T>` 必须表示圆弧段，不表示任意曲线。

### 10.1 必须保存的核心数据

第一版采用以下参数作为主表达：

- 圆心
- 半径
- 起始角
- 终止角
- 方向

执行要求：

- 第一版只允许这一套主表达
- 不允许并行维护另一套等价主表达作为核心结构

### 10.2 圆弧方向

必须显式表达圆弧方向。

必须定义：

```cpp
enum class ArcDirection
{
    Clockwise,
    CounterClockwise
};
```

不允许通过角度大小规律隐式推断方向。

### 10.3 必须满足的规则

- 能稳定返回起点
- 能稳定返回终点
- 能计算弧长
- 能计算包围盒
- 能识别无效圆弧

### 10.4 无效弧段示例

以下情况必须判为无效：

- 半径小于等于零
- 起始角与终止角表达不完整
- 圆心或参数非法

第一版不支持整圆作为单一弧段，统一只支持有明确起点和终点的有限圆弧段。

## 11. 参数化规则

所有派生段类都必须支持统一参数化：

- 参数区间固定为 `[0, 1]`

统一要求：

- `GetPointAt(0)` 返回起点
- `GetPointAt(1)` 返回终点

中间值的解释：

- 对直线段，按线性插值
- 对弧线段，按弧参数映射

这条规则后续必须贯穿：

- 投影结果
- 最近点结果
- 相交结果
- 裁剪结果

## 12. 有效性规则

所有段对象都必须支持 `IsValid()`。

统一原则：

- 构造出对象不代表它一定有效
- 使用前可显式校验
- 算法层可以在必要时提前拒绝无效对象

### 12.1 直线段无效条件

- 起点与终点重合或接近重合

### 12.2 圆弧段无效条件

- 半径非法
- 起终角非法
- 无法构成有限圆弧

具体容差规则由后续谓词模块统一定义。

## 13. 算法层如何依赖线段基类

算法层对线段体系的依赖必须分成两层。

### 13.1 公共层算法

面向 `Segment<T>` 的公共接口工作，例如：

- 读取端点
- 读取包围盒
- 读取长度
- 获取参数点

### 13.2 具体类型算法

需要几何细节时，先分发类型，再进入具体实现，例如：

- 直线段与直线段求交
- 直线段与弧线段求交
- 弧线段与弧线段求交

执行要求：

- 不要求所有算法都只靠基类虚函数解决
- 允许“统一抽象 + 显式分发”的方式

这比试图把所有算法都塞进基类虚函数里更适合工程几何库。

## 14. 第一版不做的内容

第一版线段体系不做以下内容：

- 样条曲线段
- 椭圆弧段
- 贝塞尔曲线段
- 整圆边段
- 复合曲线段
- 业务层构件边

第一版范围严格限制为：

- 有限直线段
- 有限圆弧段
- 统一边段抽象

## 15. 第一版实现清单

后续开发必须至少完成以下内容。

### 15.1 类型

- `SegmentKind`
- `ArcDirection`
- `Segment<T>`
- `LineSegment<T>`
- `ArcSegment<T>`

### 15.2 基类公共接口

- `GetKind()`
- `GetStartPoint()`
- `GetEndPoint()`
- `Length()`
- `GetBoundingBox()`
- `GetPointAt(parameter)`
- `IsValid()`

### 15.3 直线段实现

- 起点与终点存储
- 长度计算
- 起终点返回
- 参数点求值
- 包围盒计算
- 有效性判断

### 15.4 弧线段实现

- 圆心、半径、起止角、方向存储
- 起终点返回
- 弧长计算
- 参数点求值
- 包围盒计算
- 有效性判断

## 16. 开发验收标准

后续实现完成后，必须满足以下验收标准：

- 直线段与弧线段可通过统一基类抽象使用
- 调用方可以可靠区分段类型
- 所有段都具备统一起点、终点、长度、包围盒、参数点接口
- 算法层可以先基于统一接口工作，再按类型细分
- 第一版只支持直线段和圆弧段，不引入额外曲线类型

## 17. 最终执行结论

后续线段体系开发必须按以下原则执行：

- 建立统一边段基类 `Segment<T>`
- 统一支持 `LineSegment<T>` 与 `ArcSegment<T>`
- 统一参数区间为 `[0, 1]`
- 统一提供起点、终点、长度、包围盒、参数点、有效性接口
- 具体几何细节保留在派生类
- 算法层采用“统一抽象 + 显式分发”模式

本文档即为线段体系的实现依据。
