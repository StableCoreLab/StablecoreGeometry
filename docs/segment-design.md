# Segment 设计

## 1. 目的

本文定义当前仓库中 2D segment 体系的设计边界，作为后续维护、文档示例和 API 收口的参考基线。

## 2. 当前对象范围

当前 segment 体系包括：

- `Segment2<T>`
- `LineSegment2<T>`
- `ArcSegment2<T>`

## 3. 设计原则

### 3.1 统一边段抽象

`Segment2<T>` 用于统一表达线段与圆弧段所在的边段体系，但具体几何语义仍由派生或具体类型承担。

### 3.2 类型层只保留天然属性

适合保留为成员能力的包括：

- `Length()`
- `Bounds()`
- `StartPoint()`
- `EndPoint()`
- `PointAt(...)`

### 3.3 复杂算法走自由函数

以下能力继续放算法层：

- 相交
- 投影
- 距离
- 包含判定
- 关系判定

## 4. 当前固定结论

- segment 体系继续保留统一抽象
- 公开命名保持短名风格
- 多对象算法继续保留在自由函数层
